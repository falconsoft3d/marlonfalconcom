# ▷ Recomendaciones para servidores Odoo BIM en la nube

## ¿Qué recomienda Odoo para servidores Odoo BIM en Odoo servidores en la nube
1. Servidores Dedicados: Se recomienda servidores dedicados para Odoo BIM debido a su alta demanda de recursos. Estos servidores ofrecen un rendimiento óptimo y una mayor capacidad de procesamiento, lo que es esencial para manejar las complejidades de los modelos BIM y las integraciones con otras herramientas.

2. Memoria RAM: Se recomienda un mínimo de 32 GB de RAM para servidores Odoo BIM. Sin embargo, dependiendo del tamaño y la complejidad de los proyectos BIM, puede ser necesario aumentar la memoria a 64 GB o más para garantizar un rendimiento fluido.

3. Almacenamiento SSD: Se recomienda utilizar almacenamiento SSD para servidores Odoo BIM, ya que ofrece velocidades de lectura y escritura más rápidas en comparación con los discos duros tradicionales. Esto es crucial para manejar grandes archivos y garantizar tiempos de carga rápidos.

4. Procesadores Potentes: Se recomienda utilizar procesadores de alta gama, como los Intel Xeon o AMD EPYC, para servidores Odoo BIM. Estos procesadores ofrecen múltiples núcleos y un alto rendimiento, lo que es esencial para manejar las cargas de trabajo intensivas de BIM. Al menos 8 núcleos se recomienda para garantizar un rendimiento óptimo.

5. Ancho de Banda: Se recomienda un ancho de banda de al menos 100 Mbps para servidores Odoo BIM en la nube. Esto es importante para garantizar una conexión rápida y estable, especialmente cuando se trabaja con archivos BIM grandes y se realizan colaboraciones en tiempo real.

6. Instalación simple tipo https://nightly.odoo.com que evite el sobre uso de recursos y permita una instalación rápida y fácil, sin necesidad de configuraciones complejas.

7. 500 gb de almacenamiento para archivos, con posibilidad de ampliación según las necesidades del proyecto.

8. Respaldos automáticos diarios para garantizar la seguridad de los datos y la continuidad del negocio.

9. Evitar el uso de tareas programadas que consuman recursos excesivos , como las tareas de limpieza de base de datos o cron con ejecucion cada 5 minutos, ya que pueden afectar el rendimiento del servidor. En su lugar, se recomienda programar estas tareas durante períodos de menor actividad para minimizar el impacto en el rendimiento.

10. Evitar el uso de grandes archivos en la base de datos, mas bien transportarlos a un sistema de almacenamiento externo, como Amazon S3 o Google Cloud Storage, para reducir la carga en el servidor y mejorar el rendimiento. Normalemnte lo hacemos con Google Drive o Dropbox, pero para proyectos grandes es mejor usar un sistema de almacenamiento externo dedicado.

11. Hacer una prueba de carga contra el servidore de capacitación para determinar el rendimiento del servidor y ajustar la configuración según sea necesario.

12. En el tiempo ir limpiando los adjutno o mensajes de la base de datos para evitar que se acumulen y afecten el rendimiento del servidor.

13. Detectar procesos critico que esten muy lento para llevarlos de procesos con el ORM a procesos con SQL directo, para mejorar el rendimiento del servidor.
